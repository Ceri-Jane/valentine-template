## ğŸ”˜ Button Behaviour (Desktop vs Mobile)

This project intentionally changes how the **"No"** button behaves depending on the device.  
It creates a playful, teasing interaction while still working naturally on touchscreens.

All behaviour is handled in:

```
assets/js/app.js
```

The script automatically detects whether the device has a mouse or touch input:

```js
window.matchMedia("(pointer: coarse)").matches
```
---

### ğŸ–¥ï¸ <u>Desktop (mouse / trackpad)</u>

On devices with a **cursor**:

- The **No** button actively avoids being clicked
- When the cursor gets close:
  - it instantly disappears
  - a playful toast message appears
- It only reappears once the cursor moves away

Result:

- The button feels â€œuncatchableâ€
- The user naturally ends up clicking **Yes**
- Creates a fun, cheeky interaction

---

### ğŸ“± <u>Mobile / Touch devices</u>

Touchscreens donâ€™t support hover, so the behaviour changes to a tap-based system.

Instead of dodging:

- The **No** button can be tapped
- Each tap:
  - briefly hides the button
  - shows a teasing message
- After **4 taps**:
  - the button permanently transforms into **Yes**
  - it behaves exactly like the original Yes button
  - tapping it proceeds to the next page

Flow:

```
Tap 1 â†’ disappears
Tap 2 â†’ disappears
Tap 3 â†’ disappears
Tap 4 â†’ becomes YES
```

Result:

- Works naturally on touch devices
- Still playful
- Eventually leaves only â€œYesâ€ as the option ğŸ˜„
